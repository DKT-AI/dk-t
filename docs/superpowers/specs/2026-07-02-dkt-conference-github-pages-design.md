# DKT Conference — публикация на GitHub Pages + домен dk-t.dev

**Дата:** 2026-07-02

## Цель

Опубликовать готовый шаблон одностраничника конференции DKT на GitHub Pages и
привязать кастомный домен `dk-t.dev`. Сайт должен быть удобен для регулярного
редактирования (спикеры, агенда, партнёры) и следовать дизайн-системе самой
страницы DKT.

## Ключевая находка (проверено рендером)

Шаблон `DKT Conference.dc.html` — **не статичный HTML**, а экспорт из
дизайн-инструмента. Он рендерится клиентским React-рантаймом:

- `support.js` — самозагружающийся рантайм («dc-runtime»). На `DOMContentLoaded`
  подгружает React 18.3.1 + ReactDOM (по умолчанию с `unpkg.com`, с SRI-хешами —
  проверены байт-в-байт, корректны), парсит `<x-dc>`, раскрывает `<sc-if>`,
  `{{ переменные }}`, `style-hover`, рендерит в DOM.
- В standalone-режиме (`window.parent === window`) рантайм не шлёт `postMessage`
  редактору — работает автономно (проверено по коду: все `postMessage` под
  guard'ом).
- `image-slot.js` + `.image-slots.state.json` — фото спикера хранится как base64
  WebP **внутри JSON**, ездит вместе с файлом. Папка `uploads/` странице не нужна
  (в ней 4 PNG, но страница на них не ссылается).
- Живой обратный отсчёт до `2026-08-15T13:00:00+03:00` считается на клиенте;
  offset `+03:00` делает таргет корректным для любого часового пояса посетителя.

**Проверка:** локальный HTTP-сервер + headless-браузер → страница отрендерилась
полностью (hero, countdown 43 дня, спикеры, агенда, партнёр Слёрм, футер).
Единственная ошибка в консоли — отсутствующий `favicon.ico` (устраняется в Фазе 1).

**Вывод:** сайт поедет на GitHub Pages как набор статических файлов. Чтобы убрать
единственный серьёзный риск (пустая белая страница при недоступном CDN — см.
Риски), React и шрифты **селф-хостим** в репозитории (решение подтверждено).

## Дизайн-система: две разные, не смешивать

Исходник содержит папку `_ds/` — это **личный** бренд Viktor Vedmich «Deep Signal»
(teal `#14B8A6` + amber `#F59E0B`, Inter/Space Grotesk, self-hosted fonts). Его
README (стр. 21) пишет: *«Separate DKT violet brand — DO NOT MIX»*, (стр. 96)
*«green is taken by DKT»* — то есть личный бренд-док считает DKT «фиолетово-зелёным».
Реальная же конференц-страница — **сине-циан-жёлтая**. Вывод двойной:

- README подтверждает, что **`_ds/` — отдельный личный бренд**, и в публичный
  DKT-репозиторий его **не кладём**.
- Но описание DKT в этом README **устаревшее/другое**, поэтому источник токенов —
  **не README, а сама страница** (значения извлечены из `index.html`, не выдуманы).

**Подтверждено с Viktor (ревью-развилка):** «дизайн-система, которой следовать» =
**токены самой страницы DKT**, а НЕ личная система `_ds/` Deep Signal. `_ds/`
остаётся приватной.

- **Ритм:** секции `padding: 80px 0`, `max-width: 1180px`, боковой `padding: 0 32px`.

**Тон:** «встреча друзей по комьюнити», тепло, без корпоративного пафоса

## Селф-хостинг зависимостей (Фаза 1, решение подтверждено)

Убирает единственный риск «пустой белой страницы»:

- **React:** скачать `react@18.3.1/umd/react.production.min.js` и
  `react-dom@18.3.1/umd/react-dom.production.min.js` в `vendor/`. В `support.js`
  заменить `REACT_URL`/`REACT_DOM_URL` на относительные `vendor/...`. SRI для
  same-origin можно снять (или пересчитать под локальные файлы).
- **Шрифты:** заменить `<link>` на `fonts.googleapis.com` в `<head>` на
  self-hosted `@font-face` (woff2 в `assets/fonts/`) — Hanken Grotesk + JetBrains
  Mono. Убирает утечку IP в Google, FOUT и региональную недоступность.

После правок — повторный локальный рендер + скриншот (проверка, что всё грузится с
своего origin и favicon появился).

### B. Ловушки структурных правок

- **Две карточки спикеров РАЗНЫЕ.** Только первая (`dkt-speaker-1`, Alexander
  Dovnar) использует `<image-slot>`. Вторая («Загадочный гость») — хардкод с
  инлайновым `<svg>` и другими цветами. Для нового спикера **копировать карточку
  Довнара**, дать слоту **уникальный `id`** (`dkt-speaker-2`…), сменить имя/роль.
- **Фото спикера:** основной способ — положить файл в `assets/` и задать
  `src="assets/имя.jpg"` на `<image-slot>`. **Drag-and-drop работает ТОЛЬКО в
  оригинальном дизайн-редакторе** (пишет через `window.omelette`, которого на
  Pages нет) — из репозитория перетащить нельзя. Внимание: значение из
  `.image-slots.state.json` перебивает `src` — чтобы переключить слот с baked-in
  фото на файл, удалить ключ слота из JSON.
- **Hero-варианты:** в HTML лежат ОБА (центр и две колонки) в `<sc-if>`. Что
  показать — решает `heroVariant` в `data-props` + `renderVals()`, **не**
  `hint-placeholder-val` (он честен только внутри редактора). `<sc-if>`-блоки не
  трогать.
- **`data-props` — HTML-entity-encoded JSON** (`&quot;`), очень легко сломать.
  Менять дефолтные значения там опасно. Предпочтительно менять акцент через
  fallback `?? 'Бирюзовый'` в `renderVals()` и CSS-переменную; `data-props`
  считать read-only.
- **Нет билд-шага / линтера.** Единственная проверка — **отрендерить локально и
  посмотреть**. Правило: после ЛЮБОЙ правки — локальный сервер + скриншот перед
  коммитом.

## Фазы деплоя

1. **Файлы:** скопировать нужное в репо, переименовать HTML, селф-хостить React +
   шрифты (правка `support.js` + `<head>`), создать
   `.nojekyll`/`CNAME`/`CLAUDE.md`/`DESIGN.md`/`README.md`/`.gitignore`, внести 2
   правки контента. Локальный рендер + скриншот.
2. **Коммит и пуш** в `main` — **`CNAME` (ровно `dk-t.dev`, без схемы/пути/www)
   входит в первый пуш** (remote, SSH и admin-права проверены).
3. **Включить GitHub Pages** через API `POST /repos/DKT-AI/dk-t/pages`:
   `source={branch:main, path:/}`, **`build_type: "legacy"`** (branch-deploy, не
   Actions). `cname` на этом endpoint НЕ принимается — домен ставим отдельно.
4. **Кастомный домен:** `PUT /repos/DKT-AI/dk-t/pages` с `cname: "dk-t.dev"` +
   `source`. Это **идемпотентное подтверждение** уже закоммиченного `CNAME` (не
   пушить `main` без `CNAME` после этого!). GitHub запускает выпуск бесплатного
   TLS-сертификата Let's Encrypt (`.dev` в HSTS-preload → HTTPS обязателен).
5. **DNS в Squarespace** (делает Viktor вручную) — см. ниже.
6. **Проверка:** дождаться DNS → сертификат → включить «Enforce HTTPS» →
   финальный рендер на `https://dk-t.dev` (браузер + скриншот).

## Риски и крайние случаи

- **Jekyll скрывает точечные файлы** → без `.nojekyll` `.image-slots.state.json`
  (фото спикера) не отдаётся (проверено против исходника Jekyll). Обязательный файл.
  Побочно: с `.nojekyll` публичны все файлы репо (`.gitignore`, этот спек) — секретов
  в репо не держим.
- **`.dev` требует HTTPS** — работает только после выпуска сертификата GitHub;
  «Enforce HTTPS» включаем после его готовности.
- **CNAME-порядок:** свой `CNAME` коммитим ПЕРВЫМ, API-`cname` — идемпотентное
  подтверждение. Пуш `main` без `CNAME` после привязки домена молча её сбрасывает.
- **Задержка DNS + сертификата** — от минут до нескольких часов. `www→apex`-редирект
  оживает только после верификации обоих доменов — не считать временную задержку
  ошибкой.
- **CDN React** — снят селф-хостингом React в `vendor/` (иначе была бы пустая белая
  страница при недоступном unpkg, без ошибки для посетителя).
- **Отсчёт после 15 авг 2026** «замерзает» на `0/00/00/00` с подписью «До встречи
  осталось». Косметика, не блокер; пост-ивент состояние — кандидат в follow-up.
- **Пробел в имени файла** оригинала устраняется переименованием в `index.html`.
- **`new Function`/eval в рантайме:** не добавлять строгий CSP `<meta>` без
  тестирования — рантайм исполняет логику через `new Function`.
