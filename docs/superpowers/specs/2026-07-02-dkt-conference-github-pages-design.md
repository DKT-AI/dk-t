## Цель

Опубликовать готовый шаблон одностраничника конференции DKT «как есть» на GitHub
Pages и привязать кастомный домен `dk-t.dev`. Сайт должен быть удобен для
регулярного редактирования (спикеры, агенда, партнёры) и следовать дизайн-системе
самой страницы DKT.

## Ключевая находка (проверено рендером)

Шаблон `DKT Conference.dc.html` — **не статичный HTML**, а экспорт из
дизайн-инструмента. Он рендерится клиентским React-рантаймом:

- `support.js` — самозагружающийся рантайм («dc-runtime»). На `DOMContentLoaded`
  подгружает React 18.3.1 + ReactDOM с `unpkg.com` (с SRI-хешами), парсит `<x-dc>`,
  раскрывает `<sc-if>`, `{{ переменные }}`, `style-hover`, рендерит в DOM.
- В standalone-режиме (`window.parent === window`) рантайм не шлёт `postMessage`
  редактору — работает автономно.
- `image-slot.js` + `.image-slots.state.json` — фото спикера хранится как base64
  WebP **внутри JSON**, ездит вместе с файлом. Папка `uploads/` странице не нужна.
- Живой обратный отсчёт до `2026-08-15T13:00:00+03:00` считается на клиенте.

**Проверка:** локальный HTTP-сервер + headless-браузер → страница отрендерилась
полностью (hero, countdown 43 дня, спикеры, агенда, партнёр Слёрм, футер).
Единственная ошибка в консоли — отсутствующий `favicon.ico` (устраняется в Фазе 1).

**Вывод:** сайт поедет на GitHub Pages «как есть» как набор статических файлов.
Требуется интернет у посетителя для CDN React (это норма для такого экспорта).

(teal `#14B8A6` + amber, Inter/Space Grotesk). Его README прямо предупреждает:
«DKT — отдельный бренд, НЕ МЕШАТЬ». Поэтому:

- Личную `_ds/` в публичный репозиторий **не кладём**.
- Источник правды для правок этого сайта — **токены самой страницы DKT**
  (см. `DESIGN.md`, значения извлечены из `index.html`, не выдуманы).

## Модель регулярного редактирования (CLAUDE.md)

`CLAUDE.md` в корне — рабочая инструкция «как менять контент». Секции размечены
HTML-комментариями `<!-- ===== SECTION ===== -->`, что делает навигацию простой:

| Что меняем | Секция в index.html | Как |
|---|---|---|
| Спикеры | `SPEAKERS` (~123–158) | добавить карточку-копию; фото — перетащить в редакторе (пишется в `.image-slots.state.json`) или задать `src` |
| Агенда | `PROGRAM` (~160–210) | добавить/переименовать строку доклада, слот времени |
| Партнёры | `PARTNERS` (~212–235) | добавить логотип в `assets/`, вставить блок, подпись уровня |
| Дата/отсчёт | `<script>` внизу | значение `2026-08-15T13:00:00+03:00` |
| Акцент-тема | `data-props` вверху `<script>` | Бирюзовый/Малиновый/Жёлтый |

CLAUDE.md обязательно содержит правило: **любые правки следуют DESIGN.md**
(палитра, шрифты, радиусы DKT), чтобы стиль не разъезжался.

## Фазы деплоя

1. **Файлы:** скопировать нужное в репо, переименовать HTML, создать
   `.nojekyll`/`CNAME`/`CLAUDE.md`/`DESIGN.md`/`README.md`/`.gitignore`, внести 2
   правки контента. Повторный локальный рендер + скриншот (проверка favicon и
   целостности).
2. **Коммит и пуш** в `main` (remote, SSH и admin-права проверены).
3. **Включить GitHub Pages** через API: source = branch `main`, path `/` (root),
   классический deploy-from-branch (для статики — верно, без Actions).
4. **Кастомный домен:** задать `dk-t.dev` в настройках Pages через API → GitHub
   запускает выпуск бесплатного TLS-сертификата (`.dev` в HSTS-preload → HTTPS
   обязателен).
5. **DNS в Squarespace** (делает Viktor вручную) — см. ниже.
6. **Проверка:** дождаться DNS → сертификат → включить «Enforce HTTPS» →
   финальный рендер на `https://dk-t.dev` (браузер + скриншот).

## Риски и крайние случаи

- **Jekyll скрывает точечные файлы** → без `.nojekyll` `.image-slots.state.json`
  (фото спикера) не отдаётся. Обязательный файл.
- **`.dev` требует HTTPS** — работает только после выпуска сертификата GitHub;
  «Enforce HTTPS» включаем после его готовности.
- **Задержка DNS + сертификата** — от минут до нескольких часов. Ожидаемо.
- **CDN React** — при недоступности `unpkg.com` страница не отрендерится. Норма
  для этого экспорта; при желании выносится в отдельную задачу (self-host React).
- **Пробел в имени файла** оригинала (`DKT Conference.dc.html`) устраняется
  переименованием в `index.html`.

## Критерии успеха

1. `https://dk-t.dev` открывается, отдаёт валидный TLS, «Enforce HTTPS» включён.
2. Страница рендерится полностью (hero, живой отсчёт, спикеры с фото, агенда,
   партнёр «Партнёр» / Слёрм, футер), ошибок в консоли нет.
3. `www.dk-t.dev` редиректит на apex.
4. В репозитории есть `CLAUDE.md` + `DESIGN.md`, дающие следующей сессии всё для
   безопасного редактирования контента по дизайн-системе.
5. Личный бренд `_ds/` в публичный репозиторий не попал.
